Id and Type

Python have two built-in functions, id and type, which allow you to obtain the unique identifier of an object and determine its type, respectively. The id function returns a unique identifier for the object. The type function returns the type of the object.

            
            >>>example_list = [1, 2, 3]

            >>>print(id(example_list))
            # Output: 139910819433280

            >>>print(type(example_list))  
            # Output: class 'list'
            
            
            
            >>>example_int = 5

            >>>print(id(example_int))  
            # Output: 10914496
        
            >>>print(type(example_int))  
            # Output: class 'int'

Mutable Objects

Mutable objects are those can be modified after they are created. Lists, dictionaries and sets examples of mutable objects. When you modify a mutable object, its id is the same

            
            >>>example_list = [1, 2, 3]
            >>>print(id(example_list))
            # Output: 140591631764288

            >>>example_list.append(4)
            >>>print(id(example_list))
            # Output: 140591631764288

Immutable Objects

Immutable objects are those that cannot be modified after they are created. Integers, floats, strings, and tuples are examples of immutable objects. When you modify an immutable object, its id changes

            
            >>>example_int = 5
            >>>print(id(example_int))
            # Output: 11754024

            >>>example_int += 1
            >>>print(id(example_int))
            # Output: 11754056

Why does it Matter?

Understanding the difference between mutable and immutable objects is important because it can help you avoid bugs in your code. If you are not careful, you can accidentally modify an object that you did not intend to modify, which can lead to unexpected behavior.

How Python treats mutable and immutable object

Python treats mutable and immutable objects differently. When you pass a mutable object to a function, the function can modify the object in place. When you pass an immutable object to a function, the function cannot modify the object in place.
